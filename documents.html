Document branch

From OCR to applicant-level document reasoning.

FAIR-VID treats international admissions documents as a pipeline of explicit stages rather than a hidden preprocessing step. The branch begins at tesseract-ocr.ipynb and continues through local vision-language reasoning, document reconstruction, executive summarization, reconciliation, and downstream applicant-level evidence files.

Large document sets per applicant Diverse country formats Local-first privacy Inspectable JSON artifacts
Notebook context

The notebook itself advertises the next step

The tesseract-ocr notebook is labeled as the document OCR agent and points to the next vision-language document understanding notebook. The branch is represented as a chain of components.

  • OCR extraction with layout metadata and bounding boxes.
  • Zero-shot semantic interpretation through local or cloud VLM experiments.
  • Document-level executive summarization with prompt-managed outputs.
  • Reconciliation and acceptance checks before higher-level decisions.
Branch scope

Admissions documents are heterogeneous, multilingual, and hard to standardize

The paper states that OCR alone is not enough because admissions review depends on semantic interpretation of credential type, grading rules, and context. FAIR-VID combines OCR grounding with higher-level document reasoning instead of using either branch in isolation.

Paper figure

Documents are a primary evidence source

Document images are processed in parallel with structured application data and video interviews, then merged later by the orchestration and decision layer.

Branch implementation

Notebook code and generated Python tell the same story

  • The document executive-summary script clones or updates the prompt repository automatically.
  • Outputs are saved per applicant and per application instead of disappearing inside a monolithic service.
  • Prompt files remain explicit and swappable, which supports prompt traceability and audit.
  • Filter TSV files allow deterministic subdirectory selection for controlled reruns.
Privacy and limits

Cloud VLM dependency constraints

The broader draft states this directly: cloud vision models may perform well, but public-cloud document analysis is often blocked by GDPR-related transfer concerns and operational cost. FAIR-VID treats local execution as an architectural requirement.
