Project overview

Evidence streams mapped to one process twin

The core architecture combines visual document understanding, structured application data, and asynchronous video interviews. Each evidence type is processed by dedicated notebooks that emit inspectable files: OCR output, transcripts, JSON summaries, scores, and final state-transition evidence.

From the paper

Core input categories and agent families

FAIR-VID is organized around documents, structured application records, and video interviews. The website mirrors that architecture directly instead of hiding it behind a generic project summary.

System structure

Digital twin instead of an opaque end-to-end predictor

FAIR-VID mirrors the observed admission workflow. AI agents convert unstructured evidence into structured features at specific process states, while interpretable decision models decide the next transition. This separates evidence generation from process control.

Architecture

System structure and deployment pattern

The paper defines symmetric transparency and a notebook-based deployment pattern. The same process structure is shown across actors, and the same notebook decomposition is used as the source representation for later service deployment.

Symmetric process visibility

The process is represented as explicit process steps rather than a single score. The paper uses this structure to align internal workflow visibility with applicant-facing visibility.

Notebook-based digital twin implementation

Each component can be developed, executed, and audited as a notebook first. FAIR-VID then uses prompt templates and generated Python scripts to move that logic into a local or server environment.

Scope note

The paper demo focuses on methodology. The server-side FAIR-VID environment also includes additional data sources such as emails and operational side artifacts.
