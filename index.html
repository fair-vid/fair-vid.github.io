Architecture

Process-centric digital twin

The system mirrors the real admission workflow reconstructed from event logs. AI agents are invoked only at defined process states, and decision trees control auditable transitions between states.

Agent layers

📄

Visual document understanding

OCR extraction, zero-shot vision-language understanding, OCR/VLM reconciliation, document acceptance checks, and holistic document reasoning.

🎥

Video interview analysis

Whisper transcription, representative frames, behavioral stability, transcript/content scoring, visual profiling, and language analysis.

🌳

Decision rule agents

Junction-specific decision trees transform structured evidence into explicit, human-readable process transitions.

What makes this an IS development framework?

  • Process-mined foundation: the digital twin is derived from observed admission logs rather than idealized documentation.
  • State-centric orchestration: every case is mapped to a current state, and AI agents operate only at relevant decision junctions.
  • Inspectable artifacts: each notebook writes explicit JSON, OCR, transcript, score, or report artifacts for downstream inspection.
  • Notebook-to-service path: modular notebooks serve as prototype components that can be transformed into service-oriented modules.
Evaluation protocol

Derivation of Tables 1–3

The original institutional data cannot be publicly released. The companion package therefore provides synthetic event logs, schemas, notebooks, and scripts that reproduce the evaluation pipeline without exposing personal data.

Table 1: Process replay fidelity

  • Completion rate: replayed cases reaching a terminal state / all replayed cases.
  • Action agreement: replayed transitions matching the historical next action / evaluated transitions.
  • Trace similarity: normalized similarity between historical and replayed state sequences.
  • Divergence step: mean index of the first mismatching transition.
  • Outcome agreement: final accept/reject outcome matching the historical outcome.

Table 2: Document understanding

  • Credential accuracy: correctness of extracted credential type or level against human-confirmed reference fields.
  • Math accuracy: correctness of grade/conversion-related fields against confirmed references.
  • Configurations: Gemma 3 27B on institutional GPU; Gemma 3 4B in free-tier browser; Gemma 3 4B with OCR reconciliation.

Table 3: Video interview analysis

  • WER: word error rate against manually verified interview transcripts.
  • Behavioral F1: harmonic mean of precision and recall for predefined behavioral/authenticity indicators.
  • Configurations: high-resource Whisper-large + Gemma 27B and low-resource Whisper-base + Gemma 4B.

Governance and limitations

What the paper claims — and what it does not claim

Explainability

The architecture provides process-level, evidence-level, and rule-level explainability. It does not claim that every internal operation of every foundation model is fully interpretable.

Historical bias

High agreement with historical decisions is not evidence of fairness. Decision trees trained on historical transitions may reproduce systematic institutional bias and require subgroup-level audits before deployment.

Deployment status

Notebook-to-enterprise transformation is presented as an engineering feasibility demonstration. Live production validation, monitoring, authentication, and load testing remain future work.

AI-assisted tools disclosure

AI-assisted programming tools were used to support code refactoring and notebook-to-service transformation experiments. AI-assisted writing tools may be used for language editing and readability improvements. All conceptual contributions, experimental design, results, interpretations, and final manuscript decisions remain the responsibility of the authors.
